Configuration Reference

Complete YAML Configuration Documentation for All ML Packages

Overview

Both ML packages use a YAML-based configuration system. Each package ships with a default config file and optional preset configs for different algorithm combinations. Configuration values can be overridden at initialization or at runtime.

How Configuration Works

  1. Default config is loaded automatically when the package is initialized
  2. Custom config file (optional) overrides specific keys from the default
  3. Runtime overrides via config.set(key, value) take highest priority

Token Replacement

Config files use tokens that are replaced at runtime:

  • <USER_DIRECTORY_TOKEN> — replaced by data_directory_path passed to the pipeline function
  • %price_data_dir% — replaced by the resolved price_data_dir value

Example:

# In YAML config:
price_data_dir: "<USER_DIRECTORY_TOKEN>/retail_pricing/competitor_recognition_data"
store_file_path: "%price_data_dir%/Store_data_git.csv"

# At runtime with data_directory_path = "/home/user/data":
# price_data_dir becomes: "/home/user/data/retail_pricing/competitor_recognition_data"
# store_file_path becomes: "/home/user/data/retail_pricing/competitor_recognition_data/Store_data_git.csv"

Config Class API

Both packages use the same Config class interface.

from competitor_identification.config.config import Config
# or
from price_imputation_and_forecasting.config.config import Config

Config(default_config_path=None, custom_config_path=None)

Loads the default YAML config. If custom_config_path is provided, those values override the defaults.

config.load_config(config_path)

Loads a YAML file and returns the parsed dictionary.

config.update_config(config_path)

Loads a YAML file and merges its values into the current configuration (overriding existing keys).

config.get(key, default=None)

Retrieve a configuration value by key. Returns default if the key is not found.

config.set(key, value)

Set or override a configuration value at runtime.

Usage Example:

from competitor_identification import CompetitorIdentification

# Load with custom config
ci = CompetitorIdentification(custom_config_path='tsne_dbscan_config.yaml')
config = ci.get_config()

# Read a value
method = config.get('dimensionality_reduction_method')  # 't-sne'

# Override at runtime
config.set('top_n_competitors', 10)
config.set('nan_threshold', 0.3)

Competitor Identification: Data Paths

These settings control where the pipeline looks for input data files.

Key Type Default Description
price_data_dir str <USER_DIRECTORY_TOKEN>/retail_pricing/competitor_recognition_data Directory containing category Parquet files. The <USER_DIRECTORY_TOKEN> is replaced by data_directory_path at runtime.
store_file_path str %price_data_dir%/Store_data_git.csv Path to the store metadata CSV file. %price_data_dir% is resolved at runtime.
subchain_file_path str %price_data_dir%/SubChainNameEnglish.xlsx Path to the sub-chain English names Excel file.

Competitor Identification: Preprocessing

Key Type Default Description
nan_threshold float 0.25 Maximum allowed NaN ratio per row. Rows exceeding this threshold are removed. E.g., 0.25 means rows with more than 25% NaN values are dropped.
top_n_competitors int 5 Number of top competitors to return, ranked by Euclidean distance in the 4D space.

Competitor Identification: Dimensionality Reduction

Key Type Default Description
dimensionality_reduction_method str 'pca' Algorithm to use: 'pca', 't-sne', or 'umap'
dimensionality_reduction_method_params dict (varies by method) Algorithm-specific hyperparameters (see below)

PCA Parameters

Param Type Default Description
n_oversamples int 10 Oversampling parameter for randomized SVD

t-SNE Parameters

Param Type Default Description
n_components int 2 Number of output dimensions
perplexity float 40 Perplexity parameter (balances local vs global structure). Typical range: 5-50.
max_iter int 300 Maximum number of optimization iterations
method str 'exact' Gradient computation method: 'exact' or 'barnes_hut'
init str 'random' Initialization method
learning_rate float 200 Learning rate for optimization

UMAP Parameters

Param Type Default Description
n_neighbors int 5 Number of neighbors for manifold approximation. Smaller values preserve more local structure.
n_components int 2 Number of output dimensions

Competitor Identification: Clustering

Key Type Default Description
clustering_method str 'birch' Algorithm to use: 'birch', 'dbscan', or 'optics'
clustering_method_params dict (varies by method) Algorithm-specific hyperparameters (see below)

Birch Parameters

Param Type Default Description
threshold float 9 Radius of the subcluster obtained by merging a new sample. Smaller values create more clusters.
n_clusters int or null null Number of clusters after the final clustering step. null uses the sub-clusters directly.

DBSCAN Parameters

Param Type Default Description
min_samples int 5 Minimum number of points required to form a dense region
eps float or null null Maximum distance between two samples for them to be considered neighbors. null triggers automatic epsilon estimation.
n_neighbors int 2 Number of neighbors used for automatic epsilon estimation (only when eps is null)

OPTICS Parameters

Param Type Default Description
min_samples int 5 Minimum number of points in a neighborhood to define a core point
min_cluster_size float 0.02 Minimum fraction of total data points for a cluster (0-1)
metric str 'minkowski' Distance metric for neighbor calculations

Competitor Identification: Config Presets

The package ships with three pre-built YAML configuration files:

pca_birch_config.yaml (Default)

Fastest combination, good for large datasets.

# Default configurations
price_data_dir: "<USER_DIRECTORY_TOKEN>/retail_pricing/competitor_recognition_data"
store_file_path: "%price_data_dir%/Store_data_git.csv"
subchain_file_path: "%price_data_dir%/SubChainNameEnglish.xlsx"

nan_threshold: 0.25

dimensionality_reduction_method: 'pca'
dimensionality_reduction_method_params:
  n_oversamples: 10

clustering_method: 'birch'
clustering_method_params:
  threshold: 9
  n_clusters: null

top_n_competitors: 5

tsne_dbscan_config.yaml

Best visualization quality with density-based clustering.

dimensionality_reduction_method: 't-sne'
dimensionality_reduction_method_params:
  n_components: 2
  perplexity: 40
  max_iter: 300
  method: 'exact'
  init: 'random'
  learning_rate: 200

clustering_method: 'dbscan'
clustering_method_params:
  min_samples: 5
  eps: null
  n_neighbors: 2

umap_optics_config.yaml

Good balance of quality and speed with hierarchical density-based clustering.

dimensionality_reduction_method: 'umap'
dimensionality_reduction_method_params:
  n_neighbors: 5
  n_components: 2

clustering_method: 'optics'
clustering_method_params:
  min_samples: 5
  min_cluster_size: 0.02
  metric: 'minkowski'
Note: The tsne_dbscan_config.yaml and umap_optics_config.yaml files only contain the keys they override. All other settings (data paths, nan_threshold, top_n_competitors) are inherited from the default pca_birch_config.yaml.

Price Imputation & Forecasting: Data Paths

Key Type Default Description
price_data_dir str <USER_DIRECTORY_TOKEN>/retail_pricing/competitor_recognition_data Directory containing category Parquet files
store_file_path str %price_data_dir%/Store_data_git.csv Path to the store metadata CSV file
subchain_file_path str %price_data_dir%/SubChainNameEnglish.xlsx Path to the sub-chain names Excel file

Price Imputation & Forecasting: Preprocessing

Key Type Default Description
nan_threshold_ratio float 0.75 Keep observations with at least this ratio of non-NaN values. 0.75 means rows must have ≥75% valid data.
fill_nan_value float 0 Placeholder value used to replace NaN entries during masking. The model learns to predict these positions.
gap_prob float 0.3 Probability of simulating a missing gap during imputation training (data augmentation).
max_gap int 30 Maximum length of a simulated missing gap in days.
test_size float 0.2 Fraction of data reserved for testing (chronological split).
val_size float 0.2 Fraction of data reserved for validation (chronological split).
random_state int 42 Random seed for reproducibility in sliding window stride randomization.

Price Imputation & Forecasting: Sliding Window

These parameters control the sliding window mechanism used to create training samples for the forecasting model.

Key Type Default Description
lookback int 84 Number of past days used as input (historical window length). The model sees this many time steps.
horizon int 28 Number of future days to forecast (prediction horizon).
min_stride int 28 Minimum gap between the starting points of consecutive sliding windows.
max_stride int 84 Maximum gap between sliding windows. Actual stride is randomized between min and max.

Price Imputation & Forecasting: Imputation Model

The imputation model uses a bidirectional RNN architecture with a dual-input design: sequential price data (with mask) + static store features.

Key Type Default Description
imputation_model str 'bi_rnn' Bidirectional RNN architecture: 'bi_rnn', 'bi_lstm', or 'bi_gru'

imputation_model_params

Param Type Default Description
seq_units int 128 Number of units in the bidirectional recurrent layer
dense_units int 128 Number of units in the dense layer (processes static features)
dropout_rate float 0.2 Dropout rate for regularization
recurrent_dropout float 0.1 Recurrent dropout rate (dropout on recurrent connections)
epochs int 30 Number of training epochs
batch_size int 64 Training batch size
learning_rate float 0.001 Adam optimizer learning rate
callbacks list or null null Optional Keras callbacks (e.g., EarlyStopping)

Price Imputation & Forecasting: Forecasting Model

The forecasting model uses a unidirectional RNN architecture for future price prediction.

Key Type Default Description
forecasting_model str 'gru' Unidirectional RNN architecture: 'rnn', 'lstm', or 'gru'

forecasting_model_params

The recurrent layer key name varies by model type:

Param Type Default Description
gru_units / lstm_units / rnn_units int 256 Number of units in the recurrent layer. Use the key matching your model type.
dense_units int 128 Number of units in the dense output layer
dropout_rate float 0.15 Dropout rate for regularization
epochs int 30 Number of training epochs
batch_size int 64 Training batch size
learning_rate float 0.001 Adam optimizer learning rate
Important: The recurrent layer units key must match the model type. Use gru_units with forecasting_model: 'gru', lstm_units with 'lstm', and rnn_units with 'rnn'.

Price Imputation & Forecasting: Save Settings

Key Type Default Description
model_save_dir_imputation str ./saved_models/imputation/ Directory for saving trained imputation models. The category name is appended at runtime (e.g., ./saved_models/imputation/rice).
model_save_dir_forecasting str ./saved_models/forecasting/ Directory for saving trained forecasting models. The category name is appended at runtime.
